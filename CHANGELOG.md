### Fixed
- colonist-stats-tracker (1.54.0): **⛔ Cards blocked stopped reading 0 (caught by `__cstAudit`).** 1.46 made the WS board the source of truth for blocked-loss and `return`ed its value whenever the board was ready — but that accrual under-counts (the audit showed `wsBoard=0` against colonist's Victory-table `victory=11`/`2`), and returning it bypassed BOTH colonist's end-game exact value AND the log differential, so ⛔ read 0 all game (the `⛔ oracle: WS 0 vs est 11` console line was firing unnoticed the whole time). `blockLossOf` now treats the log differential — colonist's end-game exact when captured, else the per-round estimate — as the source of truth, and keeps the WS board as an oracle only (`console.debug` on divergence) until its robber-tile tracking in the diff is fixed. The end-game ⛔ shows colonist's exact count again. +1 test (244 suite).
- colonist-stats-tracker (1.49.0): **Resync no longer pads phantom "unknown" cards onto opponents (raised by Stan).** The 🔄 button (`deepRescrape`) used to `resetState()` then re-scrape colonist's virtualised chat log — only on-screen rows mount, so opponents' early history is lost and the shortfall reconciles to "unknown" (clean before resync, suddenly unknown after). When the WebSocket board is ready there is no gap to recover: it already holds a complete live snapshot. Resync now short-circuits to `syncFromWS()` — self re-read EXACT from the protocol (no phantom unknowns), every total reconciled to the WS count, and opponents keep their continuously-inferred breakdown instead of being wiped. The reset+rescrape path stays as the fallback only while the board isn't ready (lobby / pre-handshake). +1 test (226 suite).
- colonist-stats-tracker (1.41.0): **rolls and cards no longer go missing from a fast log (raised by Stan: "少算一次擲骰", inconsistent resync).** A virtual log row can mount as an empty shell and receive its text a frame later; the reader marked it "read" during the shell and then skipped it forever — dropping the occasional roll, padding a missed `got` as "unknown", and making repeated re-reads disagree. It now commits a row to the dedup set only once it actually has text, and the observer re-reads a row the instant content fills into it. (tests/log-completeness.test.js.)
- colonist-stats-tracker (1.40.0): **a disable-karma vote no longer triggers a re-read (raised by Stan).** The vote injects UI that momentarily empties the log container; the reader read that transient empty state as a brand-new log, wiped its dedup and kicked off a non-deterministic rescrape (numbers visibly jumped). An empty container is now treated as a continuation. (tests/log-container.test.js.)
- colonist-stats-tracker (1.39.0): **deterministic resync.** `deepRescrape` waited a fixed delay and could skip virtual rows that mounted late, so two re-reads could disagree; it now settles each viewport (scan until the visible row set stops growing) before stepping down, reading every row.
- colonist-stats-tracker (1.36.0): **⛔ Cards blocked no longer over-counts (~2×, raised by Stan — we showed 15/20 vs colonist's 7/13).** The old formula summed `global block-count × your yield`, double-counting same-number tiles across players and ignoring that the robber blocks a single tile. Replaced with a differential: per blocked round, loss = `max(0, your clean yield for that number+resource − what you actually got)`, so a player who wasn't blocked nets zero and a partially-blocked tile counts only the missing cards. Block events settle at the next roll / game end; `produces` is learned with `Math.max` so a partially-blocked round can't lower the clean baseline. (tests/blocked-loss.test.js; opus code-review signed off.)

### Added
- colonist-stats-tracker (1.34.0): **robber-blocked card loss + drag-reorder columns (both raised by Stan).** A new ⛔ Stats column shows the cards you *would* have collected but lost to the robber sitting on a tile you build on. colonist's board is a `<canvas>` (no geometry to read), so it's purely log-derived: each player's per-number yield is learned from clean (un-blocked) rolls, and the loss is the **derived** sum `blocked-count × your yield for that number+resource`, so a tile blocked before its number warmed up is credited retroactively (backfill is automatic; a tile never once rolled clean stays uncredited). The hover breaks it down per `N res ×times = cards`, and the lifetime popup aggregates block-loss too. Separately, both the Resources and Stats tables now **reorder by dragging a column's header icon** (4 px threshold to start; the whole column slides live; release to drop); order persists per view and resets with "Restore defaults", and `reconcileOrder()` keeps saved orders forward-compatible across version changes. Header icons set `draggable="false"` and cancel native `dragstart` so the browser's image/selection drag can't hijack the pointer reorder. +16 tests (153 total).

### Changed
- colonist-stats-tracker (1.33.0): **snappier collapse/fade reaction (raised by Stan).** The lag wasn't slow computation — it was the cadence we set: posture/ghost were checked on a fixed **250 ms** leading-edge throttle, so opening the Settings menu / a trade / collapsing / resuming could wait out most of that window before reacting. Replaced it with an **~80 ms trailing throttle**: an isolated change (the menu/trade you just opened) reacts right away, while colonist's constant in-game DOM churn still coalesces to at most one check per window. Two cost trims keep the higher cadence cheap: `boardHidden()` only does its "peek under the panel" style-recalc when the panel actually sits over a sample point (normally it's a corner widget, so it's skipped), and `updateGhost()` short-circuits while the panel is collapsed (no dialog/trade overlap scan when there's nothing to fade). No behaviour change, just latency. (137 tests.)

### Changed
- colonist-stats-tracker (1.31.0): **the panel now gets out of the way for ANY colonist screen, not just Settings (raised by Stan).** The Leave Game pop-up and the Settings→back Pause/Resume screens didn't collapse the panel, because each colonist dialog has its own hashed, per-deploy class that the old class-matching detector kept missing. This generalizes (and supersedes) the old class-based dialog "ghost" tier, which is removed; the trade ghost (a bottom strip that doesn't hide the board centre) stays a fade + click-through. The decision logic is unit-tested (`boardPostureAction`); the DOM probe is verified in real games. +4 tests (132 total).

- colonist-stats-tracker (1.5.0): player rows now follow **colonist's own player-panel order** (read from the right sidebar via stable data-attributes), and each player's **avatar** is shown next to their name — including custom avatars (the panel's avatar `<img>` src is used as-is; default robots get a colour circle behind them). Falls back to first-seen order with no avatar if the panel can't be read.
- colonist-stats-tracker (1.4.0): a ⟳ "reset size & position" button (top-right of the header) restores the panel to its default size, position (top-left), and appearance — expanded, sections unfolded — **without touching the tracked stats**. Handy after zooming/dragging it somewhere awkward. The default opening position is now the top-left corner.
- colonist-stats-tracker (1.3.0): the unknown/stolen-cards column is now headed by colonist's own face-down "?" card art (self-healing to the live URL); section folds (Dice Rolls / Resources) animate open/closed.
- colonist-stats-tracker (1.2.0): collapse-to-icon — the ▾ control shrinks the whole panel down to a single dice icon (the 🎲 spins on the way in, and reverses on the way out); click the icon to expand. Each section (Dice Rolls, Resources) also folds independently via its title chevron, so you can show just the dice odds.
- colonist-stats-tracker (1.1.0): **Bank-remaining tracker** — a "Bank left" row shows how many of each resource are still in the supply (19 − everyone's known holdings), so you get the per-resource counts colonist only displays against bots. Marked "≈ upper bound" while any stolen/unknown cards are in play.
- colonist-stats-tracker (1.1.0): resizable panel (drag the bottom-right corner), A−/A+ text scaling, and persistence of panel position, size, scale, and minimised state across reloads (localStorage).
- `preview.html` at the repo root — a standalone, game-free preview of the panel for fast UI iteration. Open it directly in a browser.
- jsdom UI smoke test that renders the panel and asserts its structure (28 tests total).

### Changed
- colonist-stats-tracker (1.1.0): UI redesign — parchment palette tuned to colonist's beige game UI (no more near-black panel); the dice histogram is now horizontal (2 → 12 left to right) with bars that grow upward so frequency reads by height; resources use colonist's own card icons (self-healing to the live asset URLs) instead of emoji.
- colonist-stats-tracker (1.1.1): UI polish from live feedback — palette retuned to colonist's light warm-grey side panels; removed the Reset button (an accidental click wiped a whole game) and the "Live/Waiting" status line; merged the Bank row into the Resources table so each resource icon now carries a bank-remaining badge in its top-right corner, colonist-style; larger card icons; dice columns show the percentage again (rounded to 1 dp); more inner padding so the table is not edge-to-edge.
- colonist-stats-tracker (1.2.0): the panel now "zooms" by width — dragging the right edge scales the font and the now em-based icons together, so digits never overflow the icons; removed the +/- text-scale buttons (width is the single zoom control). Dice percentages are shown as whole numbers (no decimal); removed the "badge = bank left" caption (the badges are self-explanatory); approximate bank counts (when stolen/unknown cards are in play) are flagged with a leading "~" on the badge instead.
- colonist-stats-tracker (1.3.0): collapse is now triggered by clicking the 🎲 dice glyph itself (the dedicated ▾ button is gone); resizing works in both directions again (drag any corner/edge — width still drives the zoom, height adds vertical room); removed the separate Σ total column (colonist's own dashboard already shows each player's hand size); bank badges no longer show the "~" approximate symbol.

### Fixed
- colonist-stats-tracker (1.0.1): executed player-to-player trades ("X gave A and got B from Y") were mis-handled. Because the message text contains " got ", it was caught by the generic "got" gain branch, which counted BOTH the given-away and received cards as a net gain for the actor — so a card you traded away was incremented instead of decremented, and the counterparty's side was never updated. Trades now correctly debit what each side gave and credit what each side got. Removed the dead `traded … with` branch (colonist does not emit that wording).
- colonist-stats-tracker (1.0.2): self-detection no longer infers the local player's name from a "You stole … from &lt;Victim&gt;" line, where the only coloured name is the victim. Previously, if such a line was the first one carrying a player avatar, the victim could be tagged as "self".
- colonist-stats-tracker (1.3.2): fixed "ghost trails" smearing below the panel over colonist's board — the panel is now promoted to its own GPU compositor layer (transform: translateZ(0)), which composites cleanly over the WebGL canvas. Trimmed the header card icons slightly for better proportion with the text.
- colonist-stats-tracker (1.3.1): resizing felt laggy because the panel had a base width/height CSS transition that eased every drag — removed it (drag is instant now), and the zoom font updates live on every resize tick instead of after a debounce. Folding a section no longer leaves empty space (the panel re-fits its height to content, so folding shrinks the whole panel). The collapse animation was reworked: the panel shrinks straight to the dice with no background circle, the dice grows smoothly (no size jump), and hovering the dice shows a drop-shadow.
- colonist-stats-tracker (1.0.3): Monopoly was under-counted. The play reads "X stole N &lt;resource&gt;" where N is a number in the text but the resource icon appears only once, so the actor was credited 1 instead of N. The amount is now parsed from the text, and every opponent's holding of that resource is zeroed.
- colonist-stats-tracker (1.1.1): the panel could turn into a tall empty box after being dragged larger and then minimised or scaled down. Height is now content-driven (auto) and only width is resizable/persisted, so minimise always collapses cleanly and −/+ can never blank the panel.
