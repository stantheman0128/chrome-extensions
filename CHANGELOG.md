- colonist-stats-tracker (1.95.0): **a live geometry self-audit — colonist's own production validates our corner→tile geometry every roll (raised by Stan).** Each roll the board now PREDICTS who should produce what purely from geometry (every building on a number-n tile that isn't robbed; settlement 1 / city 2) and compares it to colonist's actual `type 47` production broadcast. Conflicts are RECORDED, not acted on: a per-game `{confirms, conflicts, conflictSamples}` summary is stored in the archived game record, a `geometry audit: N ✓ / M ✗` line (with the last mismatch) shows in `__cstAudit`, and a `console.warn` fires the instant a roll disagrees. ### Fixed
- colonist-stats-tracker (infra): **`__cstWS.save()` now captures the full game log and the last full state, not just the recent frame buffer.** On a long game the 500-frame buffer drops the opening full state (and the early events), so a capture taken late was not replayable for the opponent-hand reconstruction — replaying the (3).json capture showed the recon reconciling correctly at every frame, i.e. the buffer had rolled past the over-count and lost the baseline. The dump is now `{frames, log, fullState}`, so the recon rebuilds deterministically from the events — what's needed to diagnose the intermittent opponent over-count from a future capture.
- colonist-stats-tracker (1.99.0): **the geometry self-audit (and the geometric blocked-loss) now reject INCOMPLETE geometry, not just empty geometry (Codex pass-6 P2).** `geomReady` checked that tiles and corners were non-empty and the robber tile was loaded, but not that the captured geometry was COMPLETE: a board with a phantom built corner (a building whose hex geometry never arrived — no coordinates, or its tiles missing) still passed, and `predictProduction` then under-counted that building so the audit settled a false conflict (and blocked-loss would read a wrong 0). It now also requires `cornerDiag(b).phantom === 0`. The signal is map-agnostic — "a built thing we can't place on the board", never a fixed 19/54 count — so it holds on colonist's variable maps; a real full state arrives atomically and complete, so phantom stays 0 and the audit still runs. +2 tests (the real fixture with one built corner omitted; fails without the gate).
- colonist-stats-tracker (1.98.0): **the panel no longer fades on the Victory screen, and a manual reload after a game ends now clears it (both raised by Stan).** (1) Once a game is over the board canvas is gone, so the dialog-overlap probe read the Victory screen as an overlay and ghosted (faded) the panel — exactly when you want to read the stats against colonist's own Victory table. `updateGhost` now keeps the panel solid and clickable whenever the lifecycle is ENDED. (2) The reload (🔄) button re-synced the same finished game's data, so back at the lobby / main screen it never visibly cleared anything; pressing it there (lifecycle LOBBY) now wipes the panel (a new `clearEndedGame`: reset state, reset the WS board, forget the game signature). During a live game it stays a deep re-sync, and on the Victory screen it still re-syncs so you can read the stats; an unpressed reload still leaves the last game on screen, as before. +1 test.
- colonist-stats-tracker (1.97.0): **the geometry self-audit no longer runs before the board geometry is loaded (Codex pass-5).** `auditRoll` opened a prediction on every live roll regardless of whether a full state had populated the tiles/corners, so a board started from diffs (or an empty type-4 shell) predicted `{}` for every roll while the server still produced, settling the next roll as a false conflict — a real-capture replay produced 12 of them. It now gates on `geomReady` (tiles + corners present, robber tile loaded); with no usable geometry the round is left inconclusive, not audited. +2 tests (both fail without the gate), plus dual-socket and rejected-frame chain-robustness tests. Two known false-positive sources are kept as data to interpret rather than suppressed: a bank shortage (the server can't pay full production, so actual < predicted — check the bank first if a ✗ shows) and a dropped or undecodable frame losing a `type 47`.
- colonist-stats-tracker (1.96.0): **the geometry self-audit is hardened against false conflicts (Codex pass-4).** Four ways a clean board could have been mis-flagged are closed: (1) the WS inspector now SERIALIZES each socket's incoming frames through one promise chain, so a frame whose `Blob.arrayBuffer()` resolves slower can't relay after a later one — which could otherwise land a roll's `type 47` before its `type 10` and settle the next roll against an empty actual; (2) a same-game reconnect mid-roll now DROPS the in-flight prediction in `applyFullState` rather than settling it against the empty actual the history replay never fills; (3) the audit only accepts roll-yield production (`type 47` with `distributionType === 1`, the value the golden geometry tests already pin), so a stray setup placement can't pollute a roll; (4) a roll that predicted nothing and produced nothing (a 7, or a number no building touches) is now counted as `skipped`, not a confirmation, so the ✓ tally means the geometry was actually exercised. The final roll is also no longer force-settled at game end (its `type 47` may not have relayed before the winner line, which would manufacture a ✗) — it's left uncounted instead. `__cstAudit` and the game record now read `N ✓ / M ✗ / K –`. +4 tests, including an adversarial Blob-reorder test that fails on the un-serialized path. Accepted limitations kept as documented: the compare aggregates to (player, resource), so two compensating geometry errors on same-number same-resource tiles could cancel (per-tile validation would need `type 28`), and a mid-game F5 still resets the in-game audit count.
- colonist-stats-tracker (infra): **the WebSocket inspector's Blob → decode → relay path is now under test (Codex pass-3 #3), and the store/privacy docs were refreshed to the shipped model (Codex pass-3 #4).** Every prior `ws-*` test started at the `postMessage` boundary; a new `ws-inspector-relay` test feeds real MessagePack Blobs through `ws-inspector.js` (which gained a small CommonJS export, inert in the page) and asserts the decoded frame relays intact and in arrival order across the async `Blob.arrayBuffer()` hop, end to end into `board.js`.
